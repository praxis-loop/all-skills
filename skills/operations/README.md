# operations

`operations` 用于基础设施、服务运行、监控、故障排查和事故响应。

适合放入本分类的 skill：

- Linux、Docker、Kubernetes、Nginx、Cloudflare 等运维流程。
- 日志排查、告警处理、健康检查、容量或性能诊断。
- 发布、回滚、备份、恢复、事故响应和变更检查清单。

不适合放入本分类的 skill：

- 代码修复、测试和 PR 工作流，放到 `coding`。
- 设备或应用的个人自动化，放到 `automation`。
- 只做数据提取和指标分析的流程，放到 `data`。

当前暂无 skill。涉及生产环境变更的 skill 必须写清确认点和安全边界。
