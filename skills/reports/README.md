# reports

`reports` 用于把工作过程、技术细节、业务进展整理成可读报告。

适合放入本分类的 skill：

- 日报、周报、月报、项目进展、复盘总结。
- 面向领导、客户、团队或个人归档的工作说明。
- 把技术活动转换成业务可理解表达的流程。

不适合放入本分类的 skill：

- 需要查询、聚合、清洗数据的底层流程，放到 `data`。
- 主要生成正式文档、PPT、表格或 PDF 的流程，放到 `documents`。
- 主要做资料调研和来源对比的流程，放到 `research`。
